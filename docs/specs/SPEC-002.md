## 1. 화면 개요

| 항목 | 내용 |
|------|------|
| 화면 ID | SCR-002 |
| 화면명 | 이벤트 참여 |
| 경로 | `/[eventId]` |
| 목적 | 참여자가 이름+비밀번호 확인 후 가용 시간을 선택하고 저장하는 화면 |
| 대상 | 참여자 |
| 주요 컴포넌트 | `TimeGrid`, `SaveButton`, 이름/비밀번호 확인 폼 |

---

## 2. 진입 흐름

```
URL 접속
  ↓
이벤트 데이터 로드 (GET /api/events/[eventId]) — 로딩 스피너 표시
  ↓
이름 + 비밀번호 입력 → "확인" 버튼 클릭
  ↓
POST /api/responses/confirm
  → found: true  → 기존 슬롯 로드 후 수정 모드
  → found: false → 빈 슬롯으로 신규 참여 모드
  → 401          → 비밀번호 불일치 오류 표시
  ↓
가용 시간 선택 (드래그/클릭) — 좌측 그리드 노출
  ↓
"저장" 버튼 클릭
  ↓
POST /api/responses → 응답 DB 반영
```

> 확인 버튼 클릭 전까지는 그리드가 노출되지 않으며, 확인 후 좌측에 그리드가 나타난다.

## 3. 레이아웃 (반응형)

> 반응형 기준점: **600px** (Tailwind 커스텀 브레이크포인트 `sm600`)

### 넓은 화면 (600px 이상) — 좌우 2컬럼

| 영역 | 위치 | 내용 |
|------|------|------|
| 좌측 (flex-1) | 왼쪽 | TimeGrid + 저장 버튼 (확인 후 표시) |
| 우측 (sm600:w-72) | 오른쪽 | 이름 / 비밀번호 / 확인 버튼 |

```
┌──────────────────────────────────┬────────────────────┐
│  (확인 후 노출)                   │  이름  [_________]  │
│  ← 수평 스크롤 →                  │  비번  [_________]  │
│  ┌─────┬───┬───┬───┐             │                    │
│  │     │3/20│3/21│3/22│           │  [확인]             │
│  │ 9:00│ □ │ □ │ □ │             │                    │
│  │10:00│ □ │ □ │ □ │             │                    │
│  └─────┴───┴───┴───┘             │                    │
│  [저장]                           │                    │
└──────────────────────────────────┴────────────────────┘
```

### 모바일 (< 600px) — 상하 단일 컬럼

| 순서 | 내용 |
|------|------|
| 1 | 이름 + 비밀번호 + 확인 버튼 |
| 2 | TimeGrid (확인 후 표시) |
| 3 | 저장 버튼 (확인 후 표시) |

**Tailwind 구조**
```
<div class="flex flex-col sm600:flex-row gap-6">
  <div class="flex-1"> <!-- 좌: 그리드 + 저장 (확인 후 노출) --> </div>
  <div class="sm600:w-72 flex-shrink-0"> <!-- 우: 이름/비번/확인 --> </div>
</div>
```

## 4. 컴포넌트 명세

### 4.1 이름 + 비밀번호 확인 폼

| 항목 | 내용 |
|------|------|
| 위치 | 우측 패널 (PC) / 상단 (모바일) |
| 이름 | 텍스트 필드 (최대 50자), 필수 입력 |
| 비밀번호 | 비밀번호 필드, 선택 입력 (빈 값 허용) |
| 확인 버튼 | POST /api/responses/confirm 호출 |
| 확인 후 | 이름/비번 필드 disabled, "다시 입력" 버튼 표시 |
| Enter 키 | 이름/비번 필드에서 Enter 시 확인 동작 |

### 4.2 TimeGrid

| 항목 | 내용 |
|------|------|
| 구조 | 열 = 날짜 (이벤트 생성 시 선택된 날짜), 행 = 1시간 단위 시간대 |
| 슬롯 크기 | min-w-[2.5rem] h-8 (날짜 많을 시 수평 스크롤) |
| 선택 상태 | 미선택: `bg-neutral-100`, 선택: `bg-primary` |
| 선택 방식 (PC) | `mousedown` → `mouseover` → `mouseup` 드래그 |
| 선택 방식 (Mobile) | `touchstart` → `touchmove` → `touchend` 터치 드래그 |
| 저장 타이밍 | "저장" 버튼 클릭 시 `POST /api/responses` 호출 (실시간 저장 없음) |
| 토글 방식 | 드래그 시작 슬롯 상태 기준으로 전체 드래그 구간 일괄 선택/해제 |
| 표시 조건 | 이름+비밀번호 확인 완료 후에만 노출 |

**드래그 상태 관리**
```typescript
type DragState = {
  isDragging: boolean;   // 드래그 중 여부
  isSelecting: boolean;  // true=선택, false=해제 (시작 슬롯 기준)
  startSlot: string | null;
}
```

### 4.3 SaveButton

| 항목 | 내용 |
|------|------|
| 위치 | 그리드 하단 |
| 텍스트 | "저장" |
| 비활성 조건 | 슬롯 0개 선택 시 → 토스트 안내 후 동작 |
| 로딩 상태 | 저장 중 버튼 비활성화 |

### 4.4 타임존 드롭다운

| 항목 | 내용 |
|------|------|
| UX | SCR-001 TimezoneSelector와 동일한 Searchable Select |
| 검색 | 드롭다운 열면 검색창 노출; 입력 텍스트로 실시간 필터링 |
| 외부 클릭 | 드롭다운 외부 클릭 시 자동 닫기 |

## 5. 동작 상세

### 5.1 이름+비밀번호 확인

| 항목 | 내용 |
|------|------|
| 트리거 | "확인" 버튼 클릭 또는 Enter 키 |
| 전제 조건 | 이름 필드 1자 이상 입력 |
| 동작 | `POST /api/responses/confirm` 호출 |
| 성공(found: true) | 기존 슬롯 로드 → 그리드에 반영, 수정 모드 진입 |
| 성공(found: false) | 빈 슬롯으로 신규 참여 모드 진입 |
| 실패(401) | 비밀번호 오류 인라인 표시 |

#### API 명세: `POST /api/responses/confirm`

```typescript
// Request Body
{
  eventId: string;   // 이벤트 고유 ID
  name: string;      // 참여자 이름 (1~50자)
  password: string;  // 비밀번호 (빈 문자열 허용)
}

// Response (성공, 신규)
{ found: false; slots: [] }

// Response (성공, 기존 응답)
{ found: true; slots: string[] }

// Response (401, 비밀번호 불일치)
{ error: "비밀번호가 일치하지 않습니다." }
```

### 5.2 저장 버튼 클릭

| 항목 | 내용 |
|------|------|
| 트리거 | "저장" 버튼 클릭 |
| 전제 조건 | 이름+비밀번호 확인 완료, 슬롯 1개 이상 선택 |
| 동작 | 이름 + 비밀번호 + 현재 선택된 전체 슬롯 배열로 `POST /api/responses` |
| 성공 | "저장되었습니다 ✓" 토스트 표시, 최신 데이터 갱신 |
| 실패(401) | "비밀번호가 일치하지 않습니다." 토스트 |
| 실패(기타) | "저장에 실패했습니다" 토스트 |

#### API 명세: `POST /api/responses`

```typescript
// Request Body
{
  eventId: string;   // 이벤트 고유 ID
  name: string;      // 참여자 이름 (1~50자)
  password: string;  // 비밀번호 (빈 문자열 허용)
  slots: string[];   // 선택한 슬롯 배열 (형식: "2026-03-20T09:00")
}

// Response
{ success: true }

// Response (401, 비밀번호 불일치)
{ error: "비밀번호가 일치하지 않습니다." }
```

#### API 명세: `GET /api/events/[eventId]`

```typescript
// Response
{
  id: string;
  title: string;
  dates: string[];       // YYYY-MM-DD 형식
  startTime: string;     // HH:00 형식
  endTime: string;       // HH:00 형식
  timezone: string;      // IANA 타임존 (이벤트 생성 시 설정)
  responses: Array<{
    id: string;
    name: string;
    // password: 보안상 미포함
    slots: string[];
  }>;
}
```

### 5.3 탭 전환

| 항목 | 내용 |
|------|------|
| 트리거 | "전체 현황" 탭 클릭 |
| 동작 | SCR-003 (GroupOverlay) 컴포넌트로 전환 + 최신 데이터 조회 |
| 상태 유지 | 탭 전환 시 내 선택 데이터 및 이름 입력값 유지 |

## 6. Error Handling

| # | 예외 상황 | 동작 |
|---|---------|------|
| 1 | 이름 미입력 후 확인 버튼 클릭 | "이름을 입력해 주세요" 인라인 에러 |
| 2 | 비밀번호 불일치 | "비밀번호가 일치하지 않습니다." 인라인 에러 |
| 3 | 슬롯 0개 선택 상태로 저장 | "가능한 시간을 하나 이상 선택해 주세요" 토스트 에러 |
| 4 | 응답 저장 실패 | 토스트 에러 (선택 상태 유지, 재시도 가능) |
| 5 | 이벤트 로드 실패 | SCR-004로 리다이렉트 |
| 6 | 이벤트 데이터 로딩 중 | 로딩 스피너 표시 (전체 화면 중앙) |
