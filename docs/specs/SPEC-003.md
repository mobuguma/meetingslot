## 2. 레이아웃 (반응형)

> 반응형 기준점: **600px** (Tailwind 커스텀 브레이크포인트 `sm600`)

### 넓은 화면 (600px 이상) — 그리드 + 우측 항상 노출 응답자 목록

그리드와 응답자 목록 패널이 가로로 나란히 배치된다. 응답자 패널은 슬롯 클릭 없이도 항상 보인다.

> 슬롯 클릭 후: 해당 슬롯 가능 참여자는 하이라이트, 불가능 참여자는 흐리게

## 3. 컴포넌트 명세

### 3.1 GroupOverlay

| 항목 | 내용 |
|------|------|
| 역할 | 전체 참여자 응답을 색상 그라디언트 + 인원 수로 시각화 |
| 데이터 | `GET /api/events/[eventId]` 응답의 `responses` 배열 |
| 업데이트 | 폴링 없음; 탭 전환 시 최신 데이터 조회 |
| 상단 표시 | PC: "총 N명 응답" 텍스트; Mobile: 클릭 가능 링크 (전체 응답자 모달) |

#### 슬롯 인원 수 표시

| 상태 | 배경 토큰 | 숫자 표시 | 텍스트 색 토큰 |
|------|-----------|---------|--------------|
| 0명 | `bg-white` | 미표시 | - |
| 1명 | `bg-primary-light` | `1` | `text-primary-dark` |
| 2~(n-1)명 | `bg-primary-300` → `bg-primary-500` (비율 보간) | 숫자 | `text-white` |
| 전원(n명) | `bg-primary-700` | 숫자 | `text-white` |

> 색상 토큰은 `DESIGN-GUIDE.md` 및 `tailwind.config.ts`에 정의된 값을 사용한다.

**인원 비율에 따른 색상 계산**
```typescript
// 전체 응답자 수 대비 슬롯 선택 인원 비율로 Tailwind 클래스 결정
function getSlotColorClass(count: number, total: number): string {
  if (count === 0) return 'bg-white';
  if (count === total) return 'bg-primary-700';
  const ratio = count / total; // 0~1
  if (ratio <= 0.25) return 'bg-primary-light';
  if (ratio <= 0.5) return 'bg-primary-300';
  return 'bg-primary-500';
}
```

#### 슬롯 숫자 렌더링 규칙

- 슬롯 크기: `min-w-[2.5rem] h-8`
- 숫자 위치: 슬롯 중앙 정렬 (`flex items-center justify-center`)
- 숫자 스타일: `text-xs font-semibold`
- 0명: 숫자 미렌더링 (슬롯 빈 상태)

### 3.2 응답자 목록 패널 (PC only, 항상 노출)

슬롯 클릭 시 해당 시간에 가능한 참여자를 하이라이트로 표시.

| 항목 | 내용 |
|------|------|
| 트리거 | 화면 진입 즉시 노출 (슬롯 클릭 불필요) |
| 슬롯 미선택 시 | 전체 응답자 이름 중립 색상으로 표시 |
| 슬롯 선택 시 | 가능 참여자: `bg-primary-light text-primary font-medium` + ✓ 아이콘 |
| 슬롯 선택 시 | 불가 참여자: `text-neutral-300` (흐리게) |
| 넓은 화면 (600px+) | `sm600:w-48` 우측 고정 패널 |
| 표시 정보 | 슬롯 선택 시 날짜+시간 레이블, "N명 중 M명 가능" |

**표시 내용 (슬롯 선택 후)**
```
응답자 목록
3월 20일 (목) 09:00
─────────────────
✓ 김철수    (bg-primary-light, text-primary)
  이영희    (text-neutral-300, 흐리게)
✓ 박민준    (bg-primary-light, text-primary)
─────────────────
3명 중 2명 가능
```

### 3.3 모바일 슬롯 클릭 모달

시간 슬롯 클릭 시 전체 응답자 목록과 하이라이트를 하단 모달로 표시.

| 항목 | 내용 |
|------|------|
| 트리거 | 시간 슬롯 셀 클릭 (Mobile만) |
| 데이터 | 전체 응답자 목록 + 해당 슬롯 가능 여부 |
| 위치 | 하단 슬라이드업 시트 (높이 auto, 최대 50vh) |
| 닫기 | "닫기" 버튼 또는 시트 외부 탭 |

### 3.4 모바일 전체 응답자 모달

"N명 응답" 텍스트 클릭 시 전체 응답자 이름 목록을 모달로 표시.

| 항목 | 내용 |
|------|------|
| 트리거 | "총 N명 응답" 텍스트 클릭 (Mobile만) |
| 데이터 | 전체 응답자 이름 목록 |
| 위치 | 하단 슬라이드업 시트 |
| 닫기 | "닫기" 버튼 또는 시트 외부 탭 |

## 4. 동작 상세

### 4.1 슬롯 클릭

| 항목 | 내용 |
|------|------|
| 트리거 | 그리드의 시간 슬롯 셀 클릭 |
| PC | 우측 응답자 목록 패널에서 가능/불가 하이라이트 |
| Mobile | 하단 모달 열기 (전체 응답자 + 하이라이트) |
| 재클릭 | 동일 슬롯 재클릭 시 선택 해제 (PC: 중립, Mobile: 모달 닫기) |

### 4.2 데이터 갱신

| 항목 | 내용 |
|------|------|
| 방식 | 폴링 없음 — 탭 전환 시점에만 `GET /api/events/[eventId]` 호출 |
| 탭 진입 | "전체 현황" 탭 클릭 시 최신 응답 데이터 조회 |
