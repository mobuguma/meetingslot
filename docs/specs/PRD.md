## Purpose

여러 사람이 공통으로 가능한 미팅 시간을 쉽고 빠르게 찾을 수 있도록 돕는 웹 서비스.
계정 가입 없이 링크 하나로 참여 가능하며, 시각적 그리드로 최적 시간대를 즉시 파악할 수 있다.

**배경:**
- 단체 일정 조율 시 이메일/채팅으로 하나씩 확인하는 비효율 해소
- when2meet / whenisgood 수준의 MVP로 시작, 향후 기능 확장

---

## Target Users

- **주최자 (Organizer)**: 미팅을 잡고 싶은 사람. 후보 날짜/시간 범위를 설정하고 참여자에게 링크를 공유.
- **참여자 (Participant)**: 링크를 받아 자신의 가능한 시간을 표시하는 사람.

---

## User Story

- 저는 **"미팅 주최자"**로서, **"여러 사람이 모두 가능한 시간을 찾기"** 위해 **"이벤트를 만들고 링크를 공유"**하고 싶습니다.
- 저는 **"참여자"**로서, **"내 가능한 시간을 알려주기"** 위해 **"그리드에서 시간을 클릭/드래그로 선택"**하고 싶습니다.
- 저는 **"주최자"**로서, **"가장 많은 사람이 가능한 시간을 한눈에 보기"** 위해 **"그룹 오버레이 그리드를 확인"**하고 싶습니다.

### Acceptance Criteria

1. 주최자가 이벤트 생성 시 고유 URL이 생성되어야 한다.
2. 참여자는 URL 접속 후 이름 입력과 시간 선택을 한 화면에서 완료하고 저장 버튼으로 제출할 수 있어야 한다.
3. 그룹 오버레이는 모든 참여자의 응답을 실시간(3초 폴링)으로 반영해야 한다.
4. 색상 그라디언트로 가능 인원 수를 직관적으로 표시해야 한다.
5. 슬롯 클릭 시 해당 시간에 가능한 참여자 목록이 그리드 옆에 표시되어야 한다.

---

## Key Features

### 1. 이벤트 생성

주최자가 미팅 이름, 시간 범위, 후보 날짜를 설정하면 고유 링크가 생성된다.

> - **이벤트 이름**: 필수 입력, 최대 50자
> - **시간 범위**: 시작/종료 시간 드롭다운 (1시간 단위, 기본값 09:00~18:00); 종료 시간이 시작 이전이면 오류 토스트 + 자동 보정
> - **날짜 선택**: 달력에서 클릭 또는 클릭앤드래그로 다중 선택/해제 (최대 14일); '모두 해제' 버튼 제공
> - **가용 시간 미리보기**: 선택한 날짜·시간 범위를 우측 테이블에 즉시 반영
> - **타임존**: 브라우저 자동 감지 후 기본값 설정, 수동 변경 가능 (전체 IANA 타임존 목록, "도시명 + (GMT+-N)" 형식 표기); 서버에는 UTC 기준으로 저장, UI는 사용자 선택 타임존 기준으로 표시
> - **생성 결과**: 고유 URL 생성 및 클립보드 복사 버튼 제공

### 2. 가용 시간 입력 (참여자)

참여자가 링크로 접속해 이름과 가능한 시간 슬롯을 한 화면에서 입력하고 저장 버튼으로 제출한다.

> - **이름 입력**: 필수, 최대 50자 (비밀번호 없음), 그리드 상단에 인라인 배치
> - **시간 그리드**: 열=날짜, 행=시간대 (30분 단위)
> - **선택 방식**: 클릭 또는 드래그로 다중 선택/해제
> - **저장 방식**: "저장" 버튼 클릭 시 서버에 반영 (이름 + 선택 슬롯 일괄 전송)

### 3. 그룹 가용성 확인

모든 참여자의 응답이 하나의 그리드에 색상 그라디언트로 표시된다.

> - **색상**: 흰색(0명) → 진한 색상(전원 가능)
> - **슬롯 클릭**: 슬롯 클릭 시 해당 시간에 가능한 참여자 목록을 그리드 옆 패널에 표시
> - **실시간 반영**: 3초 폴링으로 타인의 응답 즉시 반영
> - **뷰 전환**: 내 응답 입력 그리드 ↔ 그룹 오버레이 탭 전환

## User Scenarios

### Main Flow (주최자)

1. 홈페이지 진입 → 이벤트 이름 입력
2. 시간 범위 설정 (기본 09:00 ~ 18:00, 1시간 단위)
3. 달력에서 클릭/드래그로 후보 날짜 선택 (최대 14일)
4. 우측 가용 시간 미리보기 확인 (날짜·시간 선택 즉시 반영)
5. "이벤트 만들기" 버튼 클릭
6. 생성된 고유 URL 확인 및 복사
7. 참여자에게 URL 공유

### Main Flow (참여자)

1. 공유받은 URL로 접속
2. 이름 입력 (화면 상단 인라인 필드)
3. 그리드에서 가능한 시간 클릭/드래그 선택
4. "저장" 버튼 클릭 → 응답 서버에 반영
5. "전체 현황" 탭에서 그룹 오버레이 확인

### Exception Flow

1. **중복 이름 접속**: 기존 응답자와 이름이 같더라도 별개의 응답으로 처리 (기존 응답 불러오기 없음)
2. **빈 날짜 선택 없이 생성 시도**: "날짜를 하나 이상 선택해 주세요" 인라인 에러 표시
3. **종료 시간 ≤ 시작 시간 설정**: "종료 시간은 시작 시간보다 늦어야 합니다" 토스트(중앙 정렬) + 종료 시간 자동 보정
4. **응답 저장 실패 (네트워크 오류)**: "저장에 실패했습니다. 다시 시도해 주세요" 토스트 메시지
5. **존재하지 않는 이벤트 URL 접속**: "이벤트를 찾을 수 없습니다" 안내 페이지

## Screen Architecture

| 화면 ID | 화면명 | 경로 | 설명 | 비고 |
|---------|--------|------|------|------|
| SCR-001 | 홈 / 이벤트 생성 | `/` | 이벤트 생성 폼 | 주최자 시작점 |
| SCR-002 | 이벤트 참여 | `/[eventId]` | 이름 입력 + 시간 선택 그리드 | 참여자 시작점 |
| SCR-003 | 그룹 오버레이 | `/[eventId]` (탭 전환) | 전체 가용성 시각화 | SCR-002와 동일 경로, 탭 분리 |
| SCR-004 | 404 / 이벤트 없음 | `/not-found` | 잘못된 링크 안내 | |

## 데이터 보관 정책

- 이벤트는 생성일로부터 **90일** 동안 보관된다.
- 90일 경과 후 이벤트 링크 및 해당 이벤트의 모든 응답 데이터를 **완전 삭제(hard delete)** 한다.
